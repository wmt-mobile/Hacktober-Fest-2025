## 5. Course Schedule Problem

There are a total of `numCourses` courses you have to take, labeled from `0` to `numCourses - 1`.  

You are given an array `prerequisites` where `prerequisites[i] = [ai, bi]` indicates that you must take course `bi` first if you want to take course `ai`.  

For example, the pair `[0, 1]` indicates that to take course `0` you have to first take course `1`.  

Return `true` if you can finish all courses. Otherwise, return `false`.

### Example 1

**Input:**  
numCourses = 2, prerequisites = [[1,0]]


**Output:**  
true


**Explanation:**  
There are a total of 2 courses to take.  
To take course 1 you should have finished course 0. So it is possible.

---

### Example 2

**Input:**  
numCourses = 2, prerequisites = [[1,0],[0,1]]


**Output:**  
false


**Explanation:**  
There are a total of 2 courses to take.  
To take course 1 you should have finished course 0, and to take course 0 you should also have finished course 1. So it is impossible.


Ans->This problem is essentially about detecting whether there is a cycle in a directed graph, where each node represents a course, and a directed edge from node b to node a represents a prerequisite (b must be taken before a). If there is a cycle in the graph, it means that there are circular dependencies between courses, making it impossible to finish all of them.

Approach:

We can model this problem as a Topological Sorting problem. The topological sort of a directed graph is a linear ordering of its vertices such that for every directed edge u -> v, vertex u comes before vertex v. If we can perform a topological sort, it means there is no cycle and all courses can be completed.

We can use Depth First Search (DFS) or Kahn's Algorithm (BFS) to detect cycles and check if topological sorting is possible.

Steps using DFS (Depth-First Search):

Graph Representation:

We treat each course as a node and each prerequisite as a directed edge.

We can represent the courses and prerequisites as an adjacency list.

Cycle Detection with DFS:

Use three states to track the status of each course:

0 = Unvisited

1 = Visiting (currently in the DFS recursion stack)

2 = Visited (DFS has finished for this course)

During DFS traversal, if we encounter a course that is in the "visiting" state (1), a cycle is detected, and it’s impossible to finish all courses.

If DFS completes without finding a cycle, it’s possible to finish all courses.

DFS Approach Code:
def canFinish(numCourses, prerequisites):
    # Step 1: Create an adjacency list to represent the graph
    adj_list = {i: [] for i in range(numCourses)}
    
    for course, prereq in prerequisites:
        adj_list[course].append(prereq)
    
    # Step 2: Initialize visited list to track course states
    visited = [0] * numCourses  # 0 = unvisited, 1 = visiting, 2 = visited
    
    # Step 3: Define the DFS function
    def dfs(course):
        if visited[course] == 1:  # We found a cycle
            return False
        if visited[course] == 2:  # Already processed
            return True
        
        # Mark the current course as visiting
        visited[course] = 1
        
        # Visit all the prerequisites (neighbors) of this course
        for prereq in adj_list[course]:
            if not dfs(prereq):
                return False
        
        # Mark this course as visited (processed)
        visited[course] = 2
        return True
    
    # Step 4: Perform DFS on each course
    for course in range(numCourses):
        if visited[course] == 0:  # If the course hasn't been visited yet
            if not dfs(course):
                return False
    
    return True

Explanation of the Code:

Adjacency List:

We build an adjacency list adj_list to represent the graph, where adj_list[i] contains all courses that are prerequisites for course i.

Visited List:

visited is used to keep track of the state of each course.

0 means unvisited.

1 means the course is currently being visited (i.e., it's in the DFS stack).

2 means the course has been fully processed and doesn't have any cycle.

DFS Function:

For each course, we start a DFS traversal.

If we encounter a course that is currently being visited (state 1), we have found a cycle, and we return False.

If the course is already fully processed (state 2), we skip further processing for this course.

If there are no cycles and all prerequisites are processed, we mark the course as visited.

Main Loop:

We run DFS for each course in case there are disconnected subgraphs (sets of courses with no direct dependencies).

If DFS returns False for any course, it means there is a cycle, and it's impossible to finish all courses.

Time and Space Complexity:

Time Complexity: O(V + E), where V is the number of vertices (courses) and E is the number of edges (prerequisites). We visit each course once, and for each course, we traverse all its prerequisites (edges).

Space Complexity: O(V + E) for the adjacency list and the visited list.

Example Walkthrough:
Example 1:

Input:

numCourses = 2
prerequisites = [[1, 0]]


We have two courses: 0 and 1.

Course 1 depends on course 0.

The adjacency list will be:

0 -> []
1 -> [0]


We start DFS from course 0. There are no prerequisites for course 0, so it’s marked as visited.

Then, we start DFS from course 1, and course 0 is its prerequisite. Since course 0 is already visited, DFS continues successfully.

No cycles are detected, so the answer is True.

Output:

True

Example 2:

Input:

numCourses = 2
prerequisites = [[1, 0], [0, 1]]


We have two courses: 0 and 1, with a circular dependency between them.

The adjacency list will be:

0 -> [1]
1 -> [0]


Starting DFS from course 0, we visit course 1 and then try to visit course 0 again (which is still in the "visiting" state).

A cycle is detected, so the answer is False.

Output:

False

Conclusion:

The DFS approach helps efficiently detect cycles in the graph, and thus determine if it is possible to finish all courses based on the given prerequisites.
