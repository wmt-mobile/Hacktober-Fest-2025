6. Minimum Size Subarray Sum

Given an array of positive integers `nums` and a positive integer `target`, return the minimal length of a subarray whose sum is greater than or equal to `target`.  

If there is no such subarray, return `0` instead.

### Example 1

**Input:**  
target = 7, nums = [2,3,1,2,4,3]


**Output:**  
2


**Explanation:**  
The subarray `[4,3]` has the minimal length under the problem constraint.

---

### Example 2

**Input:**  
target = 4, nums = [1,4,4]


**Output:**  
1


---

### Example 3

**Input:**  
target = 11, nums = [1,1,1,1,1,1,1,1]


**Output:**  
0
Ans->To solve the "Minimum Size Subarray Sum" problem, we can approach it using the sliding window technique. The idea is to maintain a window of elements that sum to at least the target. As we move through the array, we adjust the window's left and right bounds to find the minimal subarray.

Approach:

Sliding Window:

We'll maintain two pointers (left and right) that represent the current subarray.

We’ll iterate over the array using the right pointer, adding elements to the sum.

Whenever the sum of the current subarray is greater than or equal to the target, we try to minimize the subarray length by moving the left pointer forward and reducing the sum until it's no longer greater than or equal to the target.

Conditions:

If the sum of the subarray is greater than or equal to the target, check the subarray length and update the result if it's smaller than the previous minimum length.

If no valid subarray is found (i.e., no sum meets or exceeds the target), return 0.

Solution Code:
def minSubArrayLen(target, nums):
    n = len(nums)
    left = 0
    current_sum = 0
    min_length = float('inf')  # Start with a large number to find the minimum

    for right in range(n):
        current_sum += nums[right]
        
        # Shrink the window from the left while the sum is >= target
        while current_sum >= target:
            min_length = min(min_length, right - left + 1)
            current_sum -= nums[left]
            left += 1
    
    # If min_length is updated, return it; otherwise, return 0
    return min_length if min_length != float('inf') else 0

Explanation:

Variables:

left and right are the pointers representing the current subarray.

current_sum keeps track of the sum of the elements in the current window.

min_length stores the minimum length of any subarray that satisfies the sum condition.

Sliding Window Logic:

The right pointer moves across the array to expand the window and adds the current number to current_sum.

Whenever the sum exceeds or meets the target, the left pointer is moved right to shrink the window, checking if the subarray's length is minimal.

Final Return:

If we find a subarray whose sum is greater than or equal to target, min_length will be updated. If no valid subarray is found, we return 0.

Time Complexity:

The algorithm runs in O(n) time where n is the number of elements in nums. Both left and right pointers move from the start to the end of the array exactly once.
