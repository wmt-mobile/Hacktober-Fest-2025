4. Write a logic for reversing a linked list.
->Reversing a linked list is a common problem in data structures. The idea is to reverse the direction of the pointers so that the head becomes the tail and vice versa. Below is the step-by-step approach and logic for reversing a singly linked list.

Singly Linked List:

A singly linked list consists of nodes, where each node has:

data: The value of the node.

next: A pointer to the next node in the list (or null if it's the last node).

Approach:

To reverse the linked list, you need to:

Maintain three pointers: prev, current, and next.

Traverse the list, and for each node, change the next pointer to point to the previous node.

Move the prev and current pointers forward until the entire list is reversed.

After the loop, the prev pointer will point to the new head of the list.

Code Implementation (in Python):
class ListNode:
    def __init__(self, value=0, next=None):
        self.value = value
        self.next = next

def reverse_linked_list(head):
    # Initialize previous and current pointers
    prev = None
    current = head
    
    # Traverse the linked list
    while current is not None:
        # Save the next node
        next_node = current.next
        
        # Reverse the current node's pointer
        current.next = prev
        
        # Move pointers one position ahead
        prev = current
        current = next_node
    
    # After the loop, prev will be the new head
    return prev

# Helper function to print the linked list
def print_linked_list(head):
    current = head
    while current:
        print(current.value, end=" -> ")
        current = current.next
    print("None")

# Example usage:
# Creating a linked list: 1 -> 2 -> 3 -> 4 -> 5 -> None
head = ListNode(1)
head.next = ListNode(2)
head.next.next = ListNode(3)
head.next.next.next = ListNode(4)
head.next.next.next.next = ListNode(5)

print("Original List:")
print_linked_list(head)

# Reversing the linked list
reversed_head = reverse_linked_list(head)

print("Reversed List:")
print_linked_list(reversed_head)

Explanation of the Code:

ListNode Class:

Represents a node in the linked list with a value and a next pointer.

reverse_linked_list Function:

Input: A head pointer to the first node of the linked list.

Output: The new head pointer of the reversed linked list.

Logic:

prev is initialized as None because the new head (which is the last node in the original list) will point to None.

current points to the current node being processed (starts at the head).

In the while loop, we keep:

Storing the next_node (the next node of the current node) to not lose track of the rest of the list.

Reversing the pointer of the current node (current.next = prev).

Moving the prev pointer to the current node and the current pointer to the next node (current = next_node).

After the loop, the prev pointer becomes the new head of the reversed list.

Helper Function print_linked_list:

Traverses the linked list and prints each node's value, helping visualize the list before and after reversal.

Example Execution:

For an input linked list:

1 -> 2 -> 3 -> 4 -> 5 -> None


After applying the reverse_linked_list function, the output will be:

5 -> 4 -> 3 -> 2 -> 1 -> None

Time Complexity:

O(n): The algorithm visits each node exactly once, where n is the number of nodes in the linked list.

Space Complexity:

O(1): The space complexity is constant because we only use a few extra pointers (prev, current, and next_node) regardless of the size of the list.

This approach works for both non-empty and empty linked lists.
